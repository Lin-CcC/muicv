# Mui简历产品上下文

## 产品定位

Mui简历帮助求职者把零散经历整理成可长期复用的职业素材库，并基于具体岗位生成、评审和导出简历。普通用户不需要理解 AI agent、skill、API 或模型配置；第一次进入时只需要知道：下载桌面 app，把现有简历或一段经历放进来，然后开始和 Mui 一起整理。

## 核心用户

- 主用户：正在求职、转岗或准备跳槽的普通求职者。可能熟悉 ChatGPT，但不熟悉命令行、Claude Code、Codex、BYOK 或 token 计费。
- 次用户：已经在用 Claude Code / Codex / Cursor 等 AI agent 的开发者和高频工具用户。他们可以走 skill 路径，但不是网站首屏主线。
- 付费/高级用户：需要更多云端模型额度、PDF 导出、岗位抓取或自带模型余额的用户。相关概念应在需要时出现，不提前压到首次体验里。

## 首次体验目标

首次体验的 aha moment 是：用户已经把第一份职业素材放进 Mui，并知道下一步可以继续聊。不要把首次体验设计成产品说明会，也不要要求用户先理解所有模块。

成功标准：

- 网站首屏 2 秒内能看懂“这是帮我整理和迭代简历的桌面 app”。
- 用户能明确知道下一步是下载 app，而不是在下载、注册、Dashboard、skill、agent 之间犹豫。
- app 登录后 3 分钟内能进入第一段“记录职业生涯”对话。
- 空状态始终给出下一步，而不是只说“暂无内容”。

## 信息架构原则

- 主线优先：普通求职者路径是“下载 app → 导入/记录素材 → 针对岗位生成简历”。
- 高级入口后置：skill、agent、API key、BYOK、token 钱包放在高级用法、设置或计费上下文中。
- 任务语言优先：使用“导入简历”“记录经历”“针对岗位生成简历”，少用“核心能力”“工作流”“平台能力”这类抽象词。
- 真实产物优先：页面展示用户会得到的东西，例如素材库、岗位档案、简历版本、PDF，而不是泛泛的 AI 能力。

## 品牌语气

Mui 可以温暖、有陪伴感、有一点轻松，但不装可爱、不卖萌过量。求职是高压场景，文案要给用户确定感和掌控感。错误、余额不足、系统放行、权限等场景不要开玩笑。

## 反目标

- 不把首页做成 AI SaaS 式能力矩阵。
- 不在首屏解释 BYOK、token、skill 协议或 agent 生态。
- 不把 onboarding 做成强制导览或长教程。
- 不为了兜底而堆满说明，用户需要的是下一步，不是所有可能性。
