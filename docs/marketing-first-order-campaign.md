# Mui 简历第一单宣传方案

日期：2026-05-19  
事件：2026-05-18，MuiCV / Mui 简历出现第一笔订单。由于支付链路或其他原因未实际收款，但用户已经走到付费动作，这是产品验证的重要节点。

## 核心叙事

不要把重点放在“没收到钱”，而是放在“有人愿意为它付费”。这是从自嗨产品走向市场验证的第一个信号。

更重要的是，宣传时不要把 Mui 简历讲成“AI 生成简历工具”。简历制作只是用户进入求职周期的第一个入口，真正的产品方向是陪用户度过完整求职周期：整理职业素材、制作简历、匹配岗位、模拟面试、面试复盘、offer 决策、入职辅导。

主标题：

> Mui 简历昨天出了第一单。钱还没到账，但我很开心。

核心表达：

- 这不是财务意义上的第一单，而是产品意义上的第一单。
- 有人愿意为“AI 求职陪跑”付费，说明这个问题真实存在。
- Mui 简历不是另一个简历生成器，而是陪用户度过整个求职周期的 AI 求职平台。
- 简历是入口，但求职不止简历；面试准备、面试复盘、offer 判断、入职适应，同样需要持续支持。
- 接下来会把支付、首次体验、隐私同步、桌面 app 和后续求职模块打磨得更稳。

语气：

- 真诚，克制，有一点创业小里程碑的兴奋。
- 不要写成“爆单”“重大突破”“感谢所有人支持”这种过度营销口吻。
- 可以承认不完美：第一单没收到钱，反而更像真实创业记录。

CTA：

- 主 CTA：下载桌面 app，从一份旧简历或一段经历开始，让 Mui 陪你进入求职流程。
- 次 CTA：如果你习惯 Claude Code / Codex / Cursor，可以直接安装 skill。
- 博客 CTA：欢迎把真实求职流程发给我：简历、面试准备、复盘、offer 选择都可以。

昨天，Mui 简历出了第一单。

## Mui 简历不只是简历生成器

从名字看，Mui 简历很容易被理解成一个“AI 生成简历”的工具。

这确实是入口之一，也是现在最容易被用户理解的功能。但如果只停在这里，产品会被压得太窄。

求职不是一个“生成 PDF”的动作，而是一段持续数周甚至数月的周期：

- 先想清楚自己做过什么
- 把经历整理成可复用、可追溯的职业素材
- 针对不同岗位组织表达
- 准备面试和行为问题
- 练习讲项目、讲冲突、讲取舍
- 面试之后复盘哪里答得好、哪里没讲清
- 拿到 offer 后做判断、谈条件
- 入职之后适应新的团队和预期

简历只是这条链路上的第一块明显痛点。

## 我真正想做的是 AI 求职陪跑

很多求职产品把重点放在模板、排版、润色和投递效率上。这些当然有价值，但我更关心的是另一个问题：

一个人在求职过程中，能不能一直有一个懂他背景、记得他经历、不会替他编造事实、又能在关键节点给出具体帮助的助手？

所以 Mui 简历想做的是 AI 求职陪跑：

- 整理职业素材：把散落在旧简历、聊天记录、项目文档和脑子里的经历结构化
- 简历制作：针对岗位组织表达，检查事实、关键词、长度和清晰度
- 模拟面试：围绕目标岗位和个人经历练习回答
- 面试复盘：把面试问题、回答表现、遗漏点沉淀下来
- Offer 决策：比较机会、风险、成长空间和现实约束
- 入职辅导：帮助用户理解新角色、制定 30/60/90 天计划

这也是为什么我不希望 Mui 简历只被讲成“生成简历”。

生成是动作，陪跑才是价值。

## Mui 简历现在是什么

现在的 Mui 简历有两条入口。

第一条是桌面 app。用户可以导入旧简历，或者直接记录一段经历。Mui 会先帮你整理职业素材，再逐步进入简历制作、岗位匹配、面试准备和复盘。

第二条是给熟悉 AI agent 的用户。你可以把 Mui 简历的 skills 装到 Claude Code、Codex、Cursor 等工具里，在自己熟悉的工作流中管理求职材料。

无论走哪条路，我都希望它坚持几件事：

- 求职周期优先，而不是单点工具优先
- 素材优先，而不是模板优先
- 基于事实生成，不替用户编造经历
- 数据尽量留在用户自己手里
- 每个节点都帮助用户做更具体的准备

## 第一单暴露的问题

第一单没收到钱，也提醒我：产品价值被感知到，只是第一步。

真正能让用户放心付费、持续使用，还需要很多基础体验足够稳定：

- 支付流程要可靠
- 账号和授权流程要少解释
- 第一次打开 app 要更快进入正题
- 云同步和隐私边界要讲清楚，也要做扎实
- 简历之外的面试、复盘和入职模块要逐步补上

这些听起来没有“AI 生成简历”那么性感，但它们决定用户会不会信任这个产品，并且在整个求职周期里持续回来使用。

## 风险和注意事项

- 不要暗示已经有稳定收入或大量用户。
- 不要把“没收到钱”写成技术事故细节，避免暴露支付系统内部问题。
- 不要过度承诺“帮你拿 offer”。可以说“陪你更认真地准备整个求职周期”。
- 不要用“AI 自动投递”“一键拿 offer”这类低信任表达。
- 不要把早期不稳定包装成成熟产品，越真实越可信。
