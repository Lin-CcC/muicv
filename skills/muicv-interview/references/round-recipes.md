# 轮次配方（round recipes）

每个面试轮次有不同的关注点。muicv-interview 的题目设计**必须**按本轮的配方来组合，而不是一份通用题库随机抽。

5 个预设轮次：`tech-screen` / `hiring-manager` / `peer-cross` / `skip-level` / `hr`。命名跟 muicv-debrief 的 frontmatter 对齐，让用户用 debrief 复盘真实面试时能跟模拟反馈对得上。

---

## 题量基准

按本轮**时长**绑死题量映射，**不要**自己加减：

| 时长 | 题量 |
|---|---|
| 30 min | 3 |
| 45 min | 5 |
| 60 min | 6-7 |
| 90 min | 8 |

下文每个轮次给的"题量基准"是 60 min 标准。其他时长按比例线性缩放。

---

## tech-screen（技术初筛）

通常是用人方的工程师 / 招聘 recruiter 给候选人打的电话或视频，30-45 min 居多。是入门关，**只筛不录**。

**题量基准（60 min）**：5 道
- 3 道 技术广度（按 JD 技术栈关键词覆盖，每条一道）
- 1 道 编码热身（简单的 LeetCode easy 级别，10-15 min）
- 1 道 基础 BQ（"为什么离开上家" 或 "为什么投这个岗"）

**必考锚题**：让候选人讲一个简历上**最近的项目**——30 秒 elevator pitch + 跟着追问 2-3 层技术细节。这道题既考表达能力又考技术深度，是 tech-screen 的最高 ROI 题。

**不会问**：职业规划、战略视野、跨团队影响力、薪资细节、文化深度问题——这些留给后面的轮次。

**信号是什么**：面试官在评"基础够不够、能不能写代码、能不能进下一轮"。具体看：
- 简历里的项目描述是否真的能讲出来（防简历注水）
- 编码题能不能在限定时间内写出来（不要求完美，要求能跑）
- 沟通是否清晰（直接 yes/no 还是兜圈子）

**示范题**（不是题库，是推导样例）：
- "你简历里写做过 Mui CMS 项目，3 年内做这个项目最难的技术决策是什么？" → 锚题
- "JD 提到 React 18 Concurrent Mode，能讲一下 Suspense 跟 useTransition 各自适合什么场景？" → 技术广度
- "这是一道编码题：实现一个 LRU cache，你能想到几种思路？" → 编码热身

---

## hiring-manager（直接领导面）

候选人未来的直接 manager。这是**录用决策权最重**的一轮，60 min 起步。

**题量基准（60 min）**：5-6 道
- 1 道 项目深挖（基于简历 highlight）
- 1 道 技术权衡（JD 提到的栈做架构决策）
- 2 道 BQ（冲突 + 失败 各一道）
- 1 道 角色契合（你为什么投这个岗）

**必考锚题**：技术决策 BQ —— "讲一个你做过的技术决策，事后看是错的，怎么处理的"。这道题同时考技术判断 + 自我反思 + 处理事后变化的能力，是 hiring-manager 的核心区分点。

**不会问**：算法热身（已被 tech-screen 过滤）、薪资细节（HR 来谈）。

**信号是什么**：面试官在评"能不能干活 + 是不是麻烦"。具体看：
- 技术决策有没有逻辑、有没有真实经历支撑
- BQ 故事有没有 STAR、有没有量化、有没有 self-awareness
- 在追问压力下的反应（防御 vs 开放）
- 能不能跟我（manager）相处（沟通节奏、是否倾听）

**示范题**：
- "我看你简历说在 ACME Corp 主导 dashboard 重构，这个项目的 scope 是怎么定的？为什么不是渐进改造？" → 项目深挖
- "JD 里提到我们有个核心服务在做从单体到微服务的迁移，你之前做过类似的事，最关键的拆分原则是什么？" → 技术权衡
- 锚题（见上）→ 技术决策 BQ

---

## peer-cross（同级 / 交叉面）

未来同级的同事，或者跨部门的 senior+ 工程师。中高阶岗位（senior+）几乎一定有这一轮。

**题量基准（60 min）**：4-5 道
- 2 道 协作 BQ（跨团队推进 / 推动他人 / 拒绝不合理需求）
- 1 道 技术深度（同 level 同事最关心你能不能 carry）
- 1 道 反向题（"你想问我什么"）

**必考锚题**：**反向题质量本身是评分项**。peer 关心的是"我愿不愿意跟这个人共事"，反向题问得好（关心团队工作方式 / 技术债 / 决策机制）= 加分；问得烂（只问福利 / 加班）= 减分。

**不会问**：薪资（HR 来谈）、职业规划（manager 来谈）、文化适配的深层问题（hr / skip-level 来谈）。

**信号是什么**：面试官在评"我愿不愿意跟这人共事"。具体看：
- 协作故事里有没有"推动他人"的能力（不只是自己埋头干）
- 技术深度跟简历声称的 level 是否匹配（防注水）
- 反向题的质量（好问题 = 好同事）
- 整体气质（合不合得来）

**示范题**：
- "讲一个你跟另一个团队意见不合，最后达成共识的故事" → 协作 BQ
- "JD 提到你会跟数据团队合作，你简历里写过 'led BI dashboard project'，那次跨团队最大的卡点是什么？" → 简历追问的协作 BQ
- "我们组有 3 个 staff、5 个 senior，技术分歧主要靠 RFC 解决——你怎么看 RFC 这种机制？" → 技术深度 + 反向引导
- "你想问我什么？" → 锚题（反向题）

---

## skip-level（老板的老板 / VP）

候选人 manager 的 manager，或者 director / VP。**只在 staff+ / EM+ 岗位才有**这一轮。

**题量基准（60 min）**：4 道
- 1 道 战略题（你接手这个团队 / 领域 6-12 个月怎么规划）
- 1 道 影响力 BQ（你怎么 influence without authority）
- 1 道 失败复盘（最大的职业失误）
- 1 道 动机（为什么离开上家、为什么这家公司）

**必考锚题**：战略题 —— "假设你接手 [JD 描述的领域] 6 个月，你会怎么规划？要解决什么问题？怎么衡量成功？"。这是和 hiring-manager 轮的**核心区分**——hiring-manager 关心你能不能干活，skip-level 关心你能不能想清楚干什么。

**不会问**：算法、编码、低层级技术细节。skip-level 的时间不会浪费在这上面。

**信号是什么**：面试官在评"这人是 leadership material 还是熟练工"。具体看：
- 战略题答得是不是有结构（问题界定 / 优先级判断 / 可衡量的成功标准）
- 影响力故事里有没有"非权力影响"的细节（说服 / 协商 / 让步）
- 失败复盘有没有真实自我反思（不是"我没失败过"或"项目失败是别人的锅"）
- 动机讲得是不是真诚（不是套路化的"贵公司业务有想象空间"）

**示范题**：
- 锚题（见上）→ 战略题
- "讲一次你说服了完全不同意你的人——他不是你的 report，你怎么做到的？" → 影响力 BQ
- "你职业生涯最大的错误是什么？" → 失败复盘

---

## hr（人事面 / 终面）

HR 或招聘负责人。通常是**最后一轮**，重点是动机 / 文化 / 谈薪 / 入职可行性。

**题量基准（60 min）**：5-6 道
- 动机（为什么这家公司）
- 离职原因（为什么离开上家）
- 职业规划（3 年后想做什么）
- 期望薪资 + 福利
- 入职时间 + 背景调查同意
- 反向题（"你想问我什么"）

**必考锚题**：**薪资期望 + 离职原因**——这两题是 hr 轮的真正考点。薪资答得不合理（要太高 = 不能录、要太低 = 把自己卖便宜）整个 offer 就崩了。离职原因如果讲得让 HR 觉得你是"麻烦员工"，文化适配也会被打问号。

**不会问**：算法、系统设计、任何深技术问题。

**信号是什么**：面试官在评"会不会鸽 offer / 薪资能不能谈拢 / 文化合不合"。具体看：
- 动机有没有具体到这家公司（"业务方向" / "团队特征" / "技术栈"）vs 套话（"贵司有发展前景"）
- 离职原因是否有怨气、是否甩锅
- 薪资期望是否合理（跟职位 level / 市场行情对得上）
- 入职时间是否真的能配合（候选人有没有其他面试在跑）

**示范题**：
- "为什么投我们公司？" → 动机
- "为什么离开上家？" → 离职原因（必考）
- "薪资期望多少？包括哪些 component（base / bonus / RSU / 签字费）？" → 薪资期望（必考）
- "你想问我什么？" → 反向题（hr 轮反向题问福利 / 团队规模 / 工作节奏都是合理的，跟 peer-cross 不一样）

## 跨轮原则

- **不重复问**：如果用户在同一对话里串多轮，**已经被前一轮问过的题**不要在后一轮再问（特别是简历项目深挖、离职原因这种 cross-cutting 题）。skill 在轮间切换时记录"上一轮问过什么"。
- **轮次切换时显式 reset**：明确告诉用户"我现在切换成 [新角色]"，并简短停顿让用户也调整状态。不要无缝切换。
- **每轮都要有锚题**：不论时间多紧，**必考锚题不能省**——它是该轮的核心区分点。
