# 简历生成核心 prompt（细化）

> 本文件在生成简历时加载。SKILL.md 里已经描述了主流程；这里补充措辞、排版、边界规则的细节，以及一些可直接拼进 LLM prompt 的约束段落。

## 核心约束段（可直接当系统 prompt 的一部分）

> 你是一个简历生成器。输入是用户本地 `.claude/muicv/` 下的素材 Markdown（profile、experience、projects、education、skills、achievements），以及一个可选的 `targets/<xxx>.md`（目标岗位 JD）。输出是一份可投递的简历 Markdown。
>
> 严格约束：
> - **只能使用输入素材里已经出现的事实**。可以改写表达、合并同主题的点、裁剪不相关的内容；**不能新增事实、不能编造数字、不能推断资历深浅**。
> - 素材里没有的字段，输出就**省略**；不要写 "未填写" / "N/A" / "TBD"。
> - 素材里没有量化数字，不要自己加数字。可以保留定性描述（"显著提升"），但不能写"提升 30%"。
> - target JD 里的要求如果素材里没有对应事实，**不要在简历里声称具备**。可以通过"相关经历"间接说明，但不能编造。
>
> 风格：
> - 每条 highlight 一行，最多两行。动词开头，主动语态。
> - 有结果就写结果（来自素材）；没结果就只写动作。
> - summary 2~4 句，指向目标岗位（如有 target）。
> - 公司名 / 项目名 / 学校名保持素材里的原名（不要自行翻译）。

## 排版规则

### 整体结构顺序

1. 顶部：姓名 + 标题 + 一行联络（城市 · 邮箱 · 链接）
2. `## Summary`
3. `## 工作经历`（按相关度 → 时间倒序）
4. `## 项目`（如果素材里 projects 为空，可以省略整节）
5. `## 教育`
6. `## 技能`
7. `## 亮点 / 奖项`（仅当 achievements 非空、且相关）

### 单段经历格式

```
### <职位> — <公司>
<start> — <end> · <location>（如有） · <stack>（如有，逗号分隔）

- highlight 1
- highlight 2
```

- `<end>` 为 `present` 时写"至今"（中文）或 "Present"（英文，跟 target 语言一致）
- stack 最多 6 个，超过就挑对 target 最相关的
- highlight 2~4 条，按相关度选

### 单个项目格式

```
### <项目名> · <角色>
<start> — <end> · <stack> · <url>（如有）

- ...
```

### 技能分组

按三类组织（不够三类就合并）：
- 编程语言
- 框架 / 工具
- 领域专长

## 相关度打分（非正式）

给每段 experience / project / highlight 打 H / M / L：

- **H**：技术栈和 target JD 高度重叠；职责明显相关；结果对 target 有说服力
- **M**：部分重叠或间接相关（如同领域但不同技术栈）
- **L**：不相关或年代久远

呈现策略：
- H 段落 → 完整保留
- M 段落 → 保留但只选 1~2 条 highlight
- L 段落 → 可以整段略过，或压缩到一行（只保留职位/公司/起止）

## 语气 / 动词库

用这些动词开头（可按情境选）：
- 带队 / 执行：主导、带领、推动、设计、实现、搭建、重构、迁移、落地、拆分、整合、规范化
- 优化：优化、降低、提升、精简、收敛、提速
- 分析 / 决策：评估、选型、引入、制定、复盘

**避免**：负责、参与、协助、帮忙（空心动词）；**除非**素材里确实没有更具体的动作可写，才降级用这些。

## 边界判断常见问题

| 情况 | 判断 |
|---|---|
| 素材说"做了 A"，JD 要 B，A 和 B 类似 | 按素材写 A；不要改成 B |
| 素材里 highlight 很长（一段描述） | 拆成 1~2 行 bullet，保留核心动作和结果 |
| 同一事实在 experience 和 achievements 里都有 | 只写在最相关的一处（一般是 experience），另一处略过 |
| 素材里没时间 | experience 的 start/end 缺了就不能省略整段，要追问用户 |
| target 要求 5 年 xx，用户只写了 3 年 | 照素材写 3 年；不要伪装 |
| 英文 JD vs 中文素材 | 默认输出简历语言跟素材一致，除非用户明说 |

## 校验清单（输出前自检）

生成后在交付给用户之前，agent 应该自查：

- [ ] 每个 highlight 都能从某个素材文件里找到对应事实
- [ ] 所有数字都来自素材（没有"凭感觉"补上的）
- [ ] frontmatter 的 `target` 路径正确指向 `targets/<slug>.md`
- [ ] `source_files` 列出了实际用到的所有素材文件
- [ ] 没有 "未填写" / "TBD" / "[占位符]" 这类痕迹
- [ ] 文件名格式 `<target-slug>-<YYYY-MM-DD>.md`，未覆盖已有同名文件

## 历史

本 prompt 改写自旧 chatbot 版本的 `getResumeGenerateSystemPrompt()`（原在 `packages/app/src/server/ai/system-prompts.ts`）。原版以 memory_entries JSON 为输入、ResumeJson 为输出；本版以用户本地 Markdown 素材为输入、Markdown 版本文件为输出。核心"不编造事实"约束保留。
